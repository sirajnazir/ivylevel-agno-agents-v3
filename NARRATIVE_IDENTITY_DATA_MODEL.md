## Executive Summary

The identity system is built on **Jenny's 4 Pillars** framework:
- **Identity**: Demographics, background, circumstances
- **Aptitude**: Academic strengths, technical skills
- **Passion**: Interests, hobbies, energy indicators
- **Service**: Volunteering, causes, community impact

These feed into a synthesis that produces archetype, spike, and brand statement.

## 1. Brand Statement (`brand_statement`)

**What**: Public-facing "who am I" statement (1-2 sentences)

**Example**:
> "I'm an accessibility engineer passionate about removing barriers for people with disabilities through technology."

**DB Location**: `profiles.narrative_brand_statement` (canonical), `profiles.brand_statement` (legacy alias)

**Generated By**: Assessment Squad → `assess_four_pillars` task

**Relationship**: The "what" - derived from Four Pillars synthesis

---

## 2. Narrative DNA (`narrative_dna`)

**What**: Core identity theme sentence - the foundational narrative connecting all pillars

**Example**:
> "Engineering student driven by accessibility innovation and social justice"

**DB Location**: `profiles.narrative_dna`

**Generated By**: EC Squad → `analyze_ec_impact_task`

**Relationship**: More detailed than brand_statement, feeds into it

---

## 3. Narrative Themes (`narrative_themes`)

**What**: 3-5 recurring identity dimensions extracted from activities/interests

**Example**:
```json
["accessibility", "engineering", "social_impact", "mentorship", "leadership"]
```

**DB Location**: `profiles.narrative_themes` (JSONB array)

**Generated By**: EC Squad pattern extraction

**Relationship**: Support the narrative_dna, used for portfolio coherence

---

## 4. Spike

**What**: Narrow area of demonstrated excellence - the standout differentiator

**Examples**:
- "Peer-reviewed publication in computational biology"
- "Founded nonprofit serving 2,000+ students"
- "State championship swimmer + tutors teammates"

**DB Location**: `profiles.spike`, `profiles.spike_confidence`

**Generated By**: EC/Assessment analysis

**Relationship**: Core component of identity_synthesis, defines award/program strategy

---

## 5. Archetype

**What**: Student classification for coaching approach (8 predefined types)

**Types**:
1. **Constrained Gritty** - High-performing despite barriers
2. **Academic Achiever** - Strong academics, needs narrative depth
3. **Creative Innovator** - Arts/entrepreneurship path
4. **Community Leader** - Service-oriented
5. **Athlete Scholar** - Recruited athlete
6. **Legacy Connector** - Family connections
7. **Late Bloomer** - Started later, high growth
8. **Undiscovered Gem** - Untapped potential

**DB Location**: `profiles.archetype_id` (FK), `profiles.archetype_confidence`

**Generated By**: Assessment Squad classification

**Relationship**: Determines coaching strategy and award/program fit scoring

---

## 6. First Principle (`first_principle`)

**What**: Core "why" - the fundamental motivation driving the student

**Example**:
> "Believing that everyone deserves access to quality healthcare"

**DB Location**: `profiles.narrative_first_principle`

**Generated By**: Narrative synthesis from essay/activity analysis

**Relationship**: The "why" that complements brand_statement (the "what")

---

## 7. Four Pillars (`pillars`)

**What**: Jenny's framework - 4 dimensions of student identity

**Structure**:
```typescript
{
  identity: { demographics, background, circumstances, experiences },
  aptitude: { academics, skills, certifications, abilities },
  passion: { interests, hobbies, energy_indicators },
  service: { volunteering, causes, populations_served, values }
}
```

**DB Location**: `profiles.pillars` (JSONB)

**Generated By**: Assessment/EC extraction

**Relationship**: FOUNDATION - everything else derives from these

---

## 8. Identity Synthesis (`identity_synthesis`)

**What**: UNIFIED object containing all identity components

**Structure**:
```typescript
{
  archetype: string,
  archetype_confidence: number,
  spike: string,
  spike_confidence: number,
  master_narrative: string,
  pillars: FourPillarsOutput,
  themes: string[],
  confidence: number
}
```

**DB Location**: `profiles.identity_synthesis` (JSONB)

**Generated By**: EC Squad final synthesis

**Relationship**: AGGREGATOR - combines all concepts into one object for downstream agents

## Key Insight: Hierarchy of Concepts

```
FOUNDATION (Raw Data)
└── Four Pillars (Identity + Aptitude + Passion + Service)

CLASSIFICATION
├── Archetype (student type)
└── Spike (standout strength)

SYNTHESIS
├── Narrative DNA (core theme sentence)
├── Narrative Themes (3-5 recurring dimensions)
├── Brand Statement (public "who am I")
└── First Principle (core "why")

AGGREGATION
└── Identity Synthesis (unified JSON object)
```

## Common Confusions Clarified

| Often Confused | Actual Difference |
|----------------|-------------------|
| `brand_statement` vs `narrative_dna` | Brand is 1-2 sentences for public; DNA is internal theme |
| `pillars` vs `identity_synthesis` | Pillars is input (4 dimensions); Synthesis is output (includes archetype, spike, narrative) |
| `themes` vs `pillars` | Themes are 3-5 strings (labels); Pillars are 4 structured objects |
| `archetype` vs `spike` | Archetype is student TYPE (classification); Spike is specific STRENGTH |
